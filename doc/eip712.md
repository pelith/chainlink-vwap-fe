# EIP-712 簽名格式

本文件說明 VWAP-RFQ-Spot 訂單的 EIP-712  typed data 簽名格式，供前端與後端對接使用。實作與合約 `VWAPRFQSpot.sol` 完全一致。

---

## 一、Domain Separator

| 欄位 | 值 | 說明 |
|------|-----|------|
| name | `VWAP-RFQ-Spot` | 合約名稱 |
| version | `1` | 版本號 |
| chainId | 部署鏈 ID | 例如 Base Sepolia: 84532 |
| verifyingContract | 合約地址 | VWAPRFQSpot 部署位址 |

**EIP712Domain 型別字串**：
```
EIP712Domain(string name,string version,uint256 chainId,address verifyingContract)
```

---

## 二、Order 結構與型別字串

**Order 型別字串**（用於 `typeHash` 計算）：
```
Order(address maker,bool makerIsSellETH,uint256 amountIn,uint256 minAmountOut,int32 deltaBps,uint256 salt,uint256 deadline)
```

**欄位說明**：

| 欄位 | Solidity 型別 | 說明 |
|------|---------------|------|
| maker | address | Maker 錢包地址 |
| makerIsSellETH | bool | true = Maker 賣 WETH 換 USDC，false = Maker 賣 USDC 換 WETH |
| amountIn | uint256 | Maker 存入數量（最小單位字串，簽名時轉為 uint256） |
| minAmountOut | uint256 | 最少可接受數量（最小單位字串） |
| deltaBps | int32 | 價格偏移基點，10000 + deltaBps 須 > 0 |
| salt | uint256 | 隨機鹽值（用於唯一性） |
| deadline | uint256 | 到期時間（Unix timestamp，秒） |

**注意**：型別字串中為 `int32 deltaBps`，欄位順序與合約 `Order` struct 嚴格一致。

---

## 三、簽名步驟（前端實作）

1. **組裝 Domain**：使用 `name`、`version`、`chainId`、`verifyingContract`
2. **組裝 Order**：將各欄位轉成正確型別（address、bool、uint256、int32）
3. **EIP-712 typed data**：
   ```json
   {
     "types": {
       "EIP712Domain": [
         { "name": "name", "type": "string" },
         { "name": "version", "type": "string" },
         { "name": "chainId", "type": "uint256" },
         { "name": "verifyingContract", "type": "address" }
       ],
       "Order": [
         { "name": "maker", "type": "address" },
         { "name": "makerIsSellETH", "type": "bool" },
         { "name": "amountIn", "type": "uint256" },
         { "name": "minAmountOut", "type": "uint256" },
         { "name": "deltaBps", "type": "int32" },
         { "name": "salt", "type": "uint256" },
         { "name": "deadline", "type": "uint256" }
       ]
     },
     "primaryType": "Order",
     "domain": {
       "name": "VWAP-RFQ-Spot",
       "version": "1",
       "chainId": 84532,
       "verifyingContract": "0x..."
     },
     "message": {
       "maker": "0x...",
       "makerIsSellETH": true,
       "amountIn": "1000000000000000000",
       "minAmountOut": "2000000000",
       "deltaBps": 0,
       "salt": "123456789",
       "deadline": 1735689600
     }
   }
   ```
4. **使用 eth_signTypedData_v4** 簽署上述 typed data
5. **編碼**：簽名結果以 hex 字串送出（可含或不含 `0x` 前綴）

---

## 四、後端驗證流程

後端 `internal/orderbook/eip712.go` 的驗證邏輯：

1. **Domain Separator**：`keccak256(typeHash || nameHash || versionHash || chainId || verifyingContract)`
2. **Struct Hash**：`keccak256(orderTypeHash || maker || makerIsSellETH || amountIn || minAmountOut || deltaBps || salt || deadline)`  
   - 各欄位以 32-byte 左補零方式 pack
3. **Digest**：`keccak256("\x19\x01" || domainSeparator || structHash)`
4. **Recover**：ECDSA 從 digest 與 signature 還原 signer，驗證是否等於 `maker`

簽名格式：65 bytes `[R || S || V]`，V 為 0/1 或 27/28（後端會處理 27→0）。

## 五、型別對應注意事項

| API/JSON | EIP-712 型別 | 範例 |
|----------|--------------|------|
| maker | address | `"0x1234...abcd"` |
| maker_is_sell_eth | bool | `true` / `false` |
| amount_in | uint256 | `"1000000000000000000"`（字串，大數） |
| min_amount_out | uint256 | `"2000000000"` |
| delta_bps | int32 | `0`、`-50`（可為負數） |
| salt | uint256 | `"123456789"` |
| deadline | uint256 | `1735689600`（秒） |
